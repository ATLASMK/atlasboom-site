# Atlas Boom — Project Brief for Claude

Concise, technical overview so a Claude chat session can pick up this project in seconds. This is a working static site (no build step, no framework) for a US hydraulic knuckle boom crane distributor.

## What this site is

**Atlas Boom** (atlasboom.com) — Atlas Chicago LLC's distributor site for 13 hydraulic knuckle boom crane models (AT-10 through AT-120, 11 to 132 US tons). Also markets 18 local SEO service pages across 6 US states (crane rental, concrete pumping, heavy haul / lowboy transport in IL, IN, MI, WI, OH, PA).

Audience: US construction / rental / project buyers. Tone: serious, industrial, premium B2B. Contact is email-only (info@atlasboom.com), no phone.

## Stack & conventions

- **Pure HTML / CSS / JS.** No framework, no bundler, no `package.json`. Just open `index.html`.
- **Fonts** via Google Fonts: `Saira Condensed` (display) + `Geist` (body) + `Geist Mono` (numbers / specs). The legacy `index.html` still uses `Barlow Condensed` + `Inter` — that is the old theme.
- **Animation**: GSAP + ScrollTrigger via CDN for legacy index; the new preview uses a custom scroll-scrubbed canvas frame sequence.
- **Color tokens** (new theme): `--bg #0A0B0D`, `--surface #13161c`, `--amber #F5A623` (single accent), `--steel #aeb7c4`. Dark cinematic industrial.
- **No em-dashes anywhere.** Replace with periods, commas, or line breaks. (LLM tell, hard banned.)

## Current status (resume notes)

## The new hero — what makes it different

`_preview-theme.html` opens with the source video `header.mp4` extracted to 73 frames in `assets/images/frames/header/` and drawn into a full-bleed canvas tied to scroll position (NOT `video.currentTime` — that stalls; this matches the pattern already in use for model orbits). After about 72% scroll, the canvas locks on the final frame and three copy blocks reveal in sequence:

- `LIFT HEAVIER.` — top-left, rises in.
- `REACH FARTHER.` — right side, slides in from the right, framing the boom's reach direction.
- Footer block (lead, CTAs, 13 / 132 / 48 stats) — middle-left, rises in.

The middle-left zone sits over a light region of the final video frame, so its copy is intentionally near-black with a soft light halo for legibility (mobile collapses to a single stacked column with white-on-dark copy). The block has no panel and no backdrop — the crane stays fully visible.

## Conventions to keep

1. **No em-dashes (—) or en-dashes (–) anywhere visible.** Period or line break instead. Date ranges use a regular hyphen.
2. **One accent color: amber `#F5A623`.** No second accent. Saturation stays restrained.
3. **One radius family** per surface: `12px` cards, `8px` buttons, `6px` chips.
4. **Real images only.** Never invent `<div>`-based fake screenshots or hand-rolled decorative SVGs. Use the assets already in `assets/images/`.
5. **`viewport`: `min-h-[100dvh]` or equivalent**, never `100vh` (iOS Safari jumps).
6. **Numbers**: tabular figures via `Geist Mono` for specs, capacities, and load tables (`font-variant-numeric: tabular-nums`).
7. **Company name everywhere is `Atlas Chicago LLC`.** Contact: email-only `info@atlasboom.com`. No phone numbers in copy (the quote form still has a Phone field for customers).
8. **Generators are the source of truth** for model and service pages — if you change a model page directly, the next `_gen_pages.py` run overwrites it. Edit the generator or its data instead.

## What needs human review or decisions

- **Theme direction.** `_preview-theme.html` is the pilot for the overhaul. Open question: does the hero composition (two copy zones framing the crane) hold up? Is the alt-block contrast on the light video region readable enough? Once approved, the same tokens, fonts, and motion get rolled into `index.html` and both generators.
- **Two missing angles per model.** Each crane currently has 3 studio angles (`side`, `hero`, `front`). The original brief asked for 5 (`rear`, `lowangle` are missing). The catalog reference renders that source those exist under `Desktop/HSA CRANE/REFERANSLAR/<slug>-ref.png` — 26 new images to generate.
- **AT-100 and AT-120 load tables.** Catalog only has a multi-curve diagram for these two, not a table. The honest options are: leave the "available on request" placeholder, transcribe a real spec sheet when HSA provides one, or embed the actual catalog diagram as an image.

If any of those decisions land while you have the chat open, the cleanest next step is to update the generator(s), re-run, and verify in the local server before pushing.
